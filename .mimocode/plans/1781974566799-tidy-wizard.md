# Plan: Show OLT Model, Uptime, and Software Version on Web Page

## Context

User wants to parse OLT info from `display version` command during connection and display it on the web page. The command runs after connect, uses double-enter pattern. Three fields: model (first uppercase line), uptime (`uptime is ...`), software version (VRP line).

## Changes

### 1. `core/olt.py` — Add `get_olt_info()` method

New method on `OltConnection` (after `disconnect()`):

```python
def get_olt_info(self) -> dict:
    self._write("display version\r\r")
    time.sleep(0.5)
    self._write(" ")
    time.sleep(1)
    output = self._read_to_prompt(5)
    info = {"model": "", "uptime": "", "version": ""}
    lines = output.strip().split("\n")
    for line in lines:
        line = line.strip()
        if not info["model"] and re.match(r'^[A-Z0-9\-]+$', line):
            info["model"] = line
        uptime_match = re.search(r'uptime is\s+(.+)', line)
        if uptime_match:
            info["uptime"] = uptime_match.group(1).strip()
        if not info["version"] and "Version" in line:
            ver_match = re.search(r'Version\s+([\d.]+)', line)
            if ver_match:
                info["version"] = ver_match.group(1).strip()
    return info
```

### 2. `diagnose.py` — Call `get_olt_info()` after connect

Add `on_olt_info=None` callback parameter to `run_diagnosis()`. After `olt.connect()` at line 147:


CLI path unchanged — `on_olt_info=None` by default.

### 3. `web/app.py` — Stream `olt_info` event via SSE

In `worker()`, before calling `run_diagnosis()`, define callback:

```python
def on_olt_info(info):
    log_q.put({"type": "olt_info", "model": info["model"],
               "uptime": info["uptime"], "version": info["version"]})
```

Pass `on_olt_info=on_olt_info` to `run_diagnosis()`. No changes needed in `generate()` — it already forwards dicts.
