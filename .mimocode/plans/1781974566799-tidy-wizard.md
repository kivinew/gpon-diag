# Plan: Add OLT (Head Station) Selection to Web Interface

## Context

The old project (`webdiag`) has a dropdown with 22 OLT stations from `switches.py`. The current web app (`gpon-diag/web`) auto-selects the first available OLT silently via `find_available_olt()`. User wants the dropdown selector from the old project ported into the new web UI.

## Changes

### 1. `config.yaml` — Add all 22 OLTs from old project's `switches.py`
Each entry gets `name` (settlement description), `host` (IP), `port: 23`. User will add `.env` credentials per OLT.

### 2. `web/app.py` — OLT selection logic
- On GET: pass `config["olts"]` list to template
- On POST: read `olt_host` from form `<select>`, find matching OLT by host in config
- Fallback to `find_available_olt()` if nothing selected
- Store `olt_name` (from config name field) alongside `olt_host` in Diagnosis model

### 3. `web/templates/index.html` — Add OLT dropdown
- `<select>` with options: "Авто-определение" + all 22 stations as `"Name (IP)"`
- Styled for dark theme

### 4. `web/templates/result.html` — Show OLT name in header

### 5. `web/static/css/styles.css` — Select element styling

## Files to modify
- `config.yaml`
- `web/app.py`
- `web/templates/index.html`
- `web/templates/result.html`
- `web/static/css/styles.css`
