# План: Оптимизация вывода отчёта и новые правила

## Задачи

### 1. Частые падения вместо счётчика
- **Файл**: `core/parser.py` — `parse_register_info()`: уже парсит все downtimes, считает `register_falls_24h`
- **Файл**: `core/engine.py` — добавить `rule_frequent_falls`: если 2+ падений за 1 час (сравнить downtime'ы из `register_all_downtimes`), выдать warning
- **Файл**: `core/report.py` — убрать строки "Количество падений ONT: N" и "Время последнего падения"

### 2. Register-info для онлайн ONT
- Уже собирается в `core/olt.py:264-268` и парсится в `diagnose.py:189-190` ✓

### 3. Проверка ошибок на активных портах
- **Файл**: `core/engine.py` — добавить `rule_eth_port_errors`: для каждого LAN-порта с `link_state == "up"` проверить `eth_errors[lan_id]` — если FCS > 0 или bad_bytes > 0, выдать warning

### 4. Аптайм > 5 дней → рекомендовать перезагрузку
- **Файл**: `core/engine.py` — добавить `rule_long_uptime`: парсить `online_duration` (формат "N day(s)..."), если > 5 дней → warning с рекомендацией перезагрузить

### 5. Убрать из текстового вывода (если нет аномалий)
- **Файл**: `core/report.py` — убрать из `to_text()`:
  - `match_state`, `config_state` (аномалии уже в рекомендациях)
  - `service_profile_id`, `line_profile_id`
- Оставить в `to_dict()` для БД

### 6. Убрать laser bias, напряжение, температуру если нет аномалий
- **Файл**: `core/report.py` — убрать строки с laser_bias_current, supply_voltage, ont_temperature из `to_text()`
- Аномалии уже ловятся engine rules

### 7. Убрать module class
- **Файл**: `core/report.py` — убрать строку `Module class: {m.module_subtype}`

### 8. Головная станция и ONT в одну строку
- **Файл**: `core/report.py` — заменить:
  ```
  Головная станция: {olt_name}
  ONT = {address}
  ```
  на:
  ```
  Головная станция: {olt_name} | ONT = {address}
  ```

## Файлы для изменения
1. `core/engine.py` — 3 новых правила (frequent_falls, eth_port_errors, long_uptime)
2. `core/report.py` — оптимизация `to_text()`, заголовок в одну строку, убрать лишние строки
