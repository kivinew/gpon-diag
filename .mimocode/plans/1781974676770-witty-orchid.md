## Part 2: gpon-diag Feature Development

### User Requirements
- **OLT Models**: MA5608T, MA5600T (CLI may differ from MA5800)
- **Operations**: Diagnostics, Remote Management, Mass Diagnostics (by port)
- **New Features**: Search ONT by MAC, Firmware version checking

### Feature 1: MAC Address Search
**File**: `core/olt.py` + `core/parser.py`

Add method `find_ont_by_mac(mac_address)` to `OltConnection`:
- Command: `display mac-address ont 0/<S>/<P>` (iterates ports or uses `display mac-address-table all`)
- Parse output to extract F/S/P/ONT-ID from MAC match
- Add `parse_mac_table()` to `core/parser.py` if needed

### Feature 2: Firmware Version Checker
**File**: `core/engine.py` + `core/models.py`

- Add `firmware_available` field to `OntMetrics` (latest known version)
- Extend `rule_bad_firmware()` to compare against known-good versions from config
- Add `good_versions` list to `config.yaml`

### Feature 3: Remote Management Commands
**File**: `core/olt.py` (new methods)

| Command | Method | Description |
|---------|--------|-------------|
| `ont reset <F>/<S>/<P> <ID>` | `reset_ont()` | Reboot ONT |
| `ont modify <F>/<S>/<P> <ID> profile-realm ...` | `change_profile()` | Change profile |
| `ont upgrade <F>/<S>/<P> <ID> <file>` | `upgrade_ont()` | Firmware upgrade |
| `ont port vlan <F>/<S>/<P> <ID> ...` | `set_vlan()` | VLAN configuration |
| `ont delete <F>/<S>/<P> <ID>` | `delete_ont()` | Remove ONT |
| `ont add <F>/<S>/<P> sn <SN>` | `add_ont()` | Add new ONT |

**Safety**: All destructive operations require `--confirm` flag.

### Feature 4: Mass Diagnostics by Port
**File**: `diagnose.py` (new mode) + `core/olt.py`

- New CLI mode: `diagnose.py port <F>/<S>/<P> --olt "OLT-17.232"`
- Collects all ONT IDs on the port via `display ont info <F> <S> <P>` (summary)
- Runs diagnosis on each ONT
- Generates consolidated report

### Feature 5: MA5608T/MA5600T Compatibility
**File**: `core/olt.py`, `core/parser.py`

- Add `olt_model` field to config
- Conditionally adjust commands based on model (e.g., `display ont info` vs `display ont info 0`)
- Add model-specific parser variants if needed
