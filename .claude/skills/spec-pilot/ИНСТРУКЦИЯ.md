# 🧭 spec-pilot — как пользоваться

Скилл для Claude Code из видео про **6 фраз**. Ты говоришь обычными словами, что хочешь
собрать — он ведёт сборку по правильному порядку (интервью → спека → параллель → проверка) и
следит, чтобы ты не слил лимит, уйдя в параллель раньше времени.

---

## Шаг 1. Положи скилл в ПРАВИЛЬНУЮ папку

⚠️ **Это главная ошибка.** Скилл нужно класть **строго** в папку `.claude/skills/` — не в
любую папку проекта. Если положишь куда попало — Claude его **не увидит**.

**Способ 1 — для всех проектов сразу (проще всего):**

```
Windows:  C:\Users\ТВОЙ_ЮЗЕР\.claude\skills\spec-pilot\
Mac:      ~/.claude/skills/spec-pilot/
```

**Способ 2 — только для одного проекта:**

```
<корень-проекта>/.claude/skills/spec-pilot/
```

### ✅ Как понять, что путь правильный

```
.claude/
└── skills/
    └── spec-pilot/
        ├── SKILL.md       ← обязательно здесь
        ├── CHEATSHEET.md
        ├── README.md
        └── ИНСТРУКЦИЯ.md
```

> Для работы скилла важен только `SKILL.md`. Если при распаковке ZIP имя `ИНСТРУКЦИЯ.md`
> отобразилось кракозябрами — это косметика, на скилл не влияет.

### ❌ Частая ошибка

Положить папку в `src/`, в корень проекта **без** `.claude/skills/`, или просто открыть файлы
в редакторе. Так Claude скилл не найдёт.

### Нужен ли перезапуск

- Папку `.claude/skills/` ты создал **впервые** → перезапусти Claude Code (в окне-приложении
  закрой и открой заново; в CLI — выйди через `/exit` и запусти снова).
- Папка `.claude/skills/` уже была, ты просто докинул `spec-pilot` → обычно подхватывается
  без перезапуска. Не видит — тогда перезапусти.

## Шаг 2. Просто скажи, что хочешь собрать

⚠️ Английские фразы из ролика (`interview me`, `launch sub agents`…) печатать **НЕ нужно** —
это мнемоника. Пиши задачу обычными словами, скилл сам проведёт по порядку.

> Хочу бота, который принимает заявки и складывает их в таблицу.

> spec-pilot: собери лендинг под запись на консультацию.

Скилл сам: определит масштаб (на мелочь не будет гонять конвейер) → задаст пару вопросов →
соберёт спеку и **покажет на подтверждение** → только после твоего «ок» начнёт собирать → в
конце проверит, а оплату/удаление данных отдаст тебе на проверку.

Если скилл не подхватился сам — начни фразу с **`spec-pilot:`**.

### Как выглядит успешный прогон

```
Ты:    Хочу форму записи на консультацию.
Claude: (3 вопроса) Какие поля? Куда сохранять заявки? Что показать после отправки?
Ты:    Имя, телефон, дата. В Google-таблицу. «Спасибо, мы свяжемся».
Claude: Вот спека: [поля, хранилище, экран успеха, критерий готовности]. Запускаю сборку?
Ты:    Да.
Claude: (собирает по спеке) → (проверяет) → готово. Оплату не подключал — это твоя зона.
```

## Шаг 3. Шпаргалка под рукой

Файл `CHEATSHEET.md` — «6 фраз и правильный порядок» одним листом. Распечатай или держи
открытой, если хочешь вести порядок руками.

## Если не работает

| Что случилось | Что делать |
|---|---|
| Claude не видит скилл | Проверь путь `…/skills/spec-pilot/SKILL.md`; если папку создал впервые — перезапусти Claude Code |
| Скилл не включается на фразу | Скажи явно: «`spec-pilot: <задача>`» |
| Интервью задаёт слишком много вопросов | Скажи «коротко, это средняя задача» — урежет до 3 |
| Claude начал собирать без спеки | Стоп-фраза: «**стоп, сначала спека**» — вернётся на шаг спеки |
| Саб-агенты не запускаются | Норма на части сборок / на Pro. Скажи «иди последовательно по спеке» |
| «Не получил ×10 как в видео» | Так и должно быть: честный выигрыш ×3–×5, а на мелочи скилл и не включает весь порядок |
